Why distributed appears slower
  • JVM + serialisation overhead on single Colab node
  • 5-fold × 9-param = 45 Spark fits (sklearn used no CV)
  • 54K rows below distributed-advantage threshold
Why distributed is still right
  • Spark time grows sub-linearly with data size
  • PySpark RF leads on PR-AUC (0.6651 vs 0.5870)
  • Enables Structured Streaming for real-time
  • Fault-tolerant national-scale deployment
